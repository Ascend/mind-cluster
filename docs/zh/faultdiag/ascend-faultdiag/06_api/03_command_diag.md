# diag 命令（故障诊断）

## 功能说明

用于分析节点故障根因，如需分析多节点的故障根因，诊断前需要将各节点的清洗结果汇总到同一目录下，目录格式请参考[采集日志归档目录结构](../05_usage/02_log_collection.md#采集日志归档目录结构)。

## 命令格式

```shell
ascend-fd diag [-h] -i INPUT_PATH -o OUTPUT_PATH [-p] [-s {host,super_pod}]
```

## 参数说明

| 参数              | 类型   | 必选 | 说明                                                                         |
|-------------------|--------|------|------------------------------------------------------------------------------|
| -h, --help        | -      | 否   | 显示帮助信息                                                                 |
| -p, --performance | -      | 否   | 执行所有诊断模块（含设备资源分析和网络拥塞分析）                             |
| -i, --input_path  | String | 是   | 清洗完毕数据输入路径                                                         |
| -o, --output_path | String | 是   | 诊断结果输出路径                                                             |
| -s, --scene       | String | 否   | 诊断场景，可选值：`host`（主机场景）、`super_pod`（超节点场景）；默认 `host` |

诊断输入目录结构：

```text
诊断输入目录
├── 节点1清洗结果/
│   ├── ascend-rc-parser.json
│   ├── ascend-kg-analyzer.json
│   ├── ascend-kg-parser.json
│   └── ...
├── 节点2清洗结果/
│   └── ...
└── 节点N清洗结果/
    └── ...
```

> [!NOTE]
>
> - 多机清洗数据，需手动将每个节点的清洗结果合并到一个目录下。
> - 单机清洗数据结构，请参考节点1清洗结果，字段意义请参考[清洗结果说明](02_command_parse.md#清洗结果说明)。

## 诊断结果回显参数说明

<!-- markdownlint-disable -->
| 一级参数     | 二级参数   | 说明                                                                                                       |
|--------------|------------|------------------------------------------------------------------------------------------------------------|
| 根因节点分析 | -          | 用于分析故障根因节点。                                                                                     |
| -            | 根因节点   | 根因设备所在的 Device，如果为 Unknown Device，是由于日志采集不完整或日志中无有效设备信息。                                                                                     |
| -            | 现象描述   | 根因节点分析的现象描述。                                                                                   |
| -            | 首错节点   | 任务中最早发生错误的 Device。                                                                               |
| -            | 尾错节点   | 任务中最晚发生错误的 Device。                                                                               |
| -            | Plog 日志   | 当根因节点为 Unknown Device 且存在首错节点时，展示首错节点从首个 error 日志行开始的前 10 行 Plog 日志。        |
| -            | 日志说明   | 当根因节点为 Unknown Device 且存在首错节点时，展示首错节点的原始 Plog 日志的路径说明。                     |
| -            | 根因节点链 | 重传超次时故障节点的传播关系。                                                                             |
| -            | 卡间等待链 | 发生 Socket/Notify 超时故障节点的传播关系。                                                                  |
| 故障事件分析 | -          | 用于分析故障根因节点所在设备的根因错误。                                                                   |
| -            | 状态码     | <ul><li>当诊断出故障时，显示具体的故障码。</li><li>当未诊断出故障时，显示 NORMAL_OR_UNSUPPORTED。</li></ul> |
| -            | 故障名称   | 具体的故障名称。                                                                                           |
| -            | 故障分类   | 故障的类别及所在的组件和模块。                                                                             |
| -            | 故障设备   | 发生故障的设备。如果为 Unknown Device 未定位到设备，如日志中无有效设备信息。                                                                                             |
| -            | 故障描述   | 针对该故障的详细描述或说明。                                                                               |
| -            | 建议方案   | 针对该故障的处理建议。                                                                                     |
| -            | 关键日志   | 该故障对应的故障日志。                                                                                     |
| -            | 关键传播链 | 展示该故障引发关系中最长的一条链路。                                                                       |
| 设备资源分析 | -          | 用于分析设备的资源状态。                                                                                   |
| -            | 状态码     | <ul><li>当诊断出故障时，显示具体的故障码。</li><li>当未诊断出故障时，显示 NODE_DIAGNOSIS_NORMAL。</li></ul> |
| -            | 故障设备   | 发生故障所在的节点名称。                                                                                   |
| -            | 故障名称   | 具体的故障名称。                                                                                           |
| -            | 建议方案   | 针对该故障的处理建议。                                                                                     |
| 网络拥塞分析 | -          | 用于分析节点间的网络状态。                                                                                 |
| -            | 状态码     | <ul><li>当诊断出故障时，显示具体的故障码。</li><li>当未诊断出故障时，显示 NET_DIAGNOSIS_NORMAL。</li></ul>  |
| -            | 故障设备   | 发生故障所在的节点名称。                                                                                   |
| -            | 故障名称   | 具体的故障名称。                                                                                           |
| -            | 建议方案   | 针对该故障的处理建议。                                                                                     |

> [!NOTE]
>
> - 回显出现根因节点分析和故障事件分析，表示当前故障已经导致训练任务异常退出。
> - 回显未诊断出根因节点且故障事件分析无结果，才会执行设备资源分析和网络拥塞分析，表示当前故障属于性能劣化问题，不会导致训练任务异常退出。
> - 部分关键参数仅在特定场景中存在。

### diag_report.json 字段说明

- 顶层字段

<!-- markdownlint-disable -->
| 字段              | 类型   | 说明                                       |
|-------------------|--------|--------------------------------------------|
| `Version`         | String | Fault-Diag 版本号。                        |
| `Build_Time`      | String | Fault-Diag 构建时间。                      |
| `Task_id`         | String | 诊断任务 ID。                              |
| `Root_Cluster`    | Object | 根因节点分析结果，单机诊断时不输出。       |
| `Knowledge_Graph` | Object | 故障事件分析结果。                         |
<!-- markdownlint-enable -->

- Root_Cluster 字段

<!-- markdownlint-disable -->
| 字段                       | 类型                       | 说明                                                           |
|----------------------------|----------------------------|----------------------------------------------------------------|
| `analyze_success`          | Boolean                    | 根因节点分析是否成功，true 成功，false 失败。                  |
| `fault_description`        | Object                     | 故障描述。                                                     |
| `fault_description.code`   | Integer                    | 故障码。                                                       |
| `fault_description.string` | String                     | 故障码描述。                                                   |
| `root_cause_device`        | List[String]               | 根因设备列表。                                                 |
| `device_link`              | List                       | 根因节点链，重传超次时故障节点的传播关系。                     |
| `remote_link`              | String                     | 卡间等待链，发生超时故障时节点的传播关系。                     |
| `first_error_device`       | String                     | 任务中最早发生错误的 Device 及其错误时间。                     |
| `last_error_device`        | String                     | 任务中最晚发生错误的 Device 及其错误时间。                     |
| `note`                     | String                     | 备注，根因节点分析的补充说明。                                 |
| `fault_description_list`   | List                       | 故障描述列表。                                                 |
| `mindie_error_device`      | List[String]               | MindIE 建链故障设备列表。                                      |
| `show_device_info`         | Object                     | 首个根因节点或首错节点的展示信息，字段见下表。                 |
| `detect_workers_devices`   | Dict[String, List[String]] | 各 worker 下检测到的设备 ID 列表。                             |
<!-- markdownlint-enable -->

- show_device_info 字段

<!-- markdownlint-disable -->
| 字段             | 类型   | 说明                                                                                |
|------------------|--------|-------------------------------------------------------------------------------------|
| `device_type`    | String | 设备类型，取值为 `first_root_device`（首个根因节点）或 `first_error_device`（首错节点）。 |
| `device`         | String | 设备信息。                                                                          |
| `plog_file_path` | String | Plog 日志文件路径。                                                                 |
| `error_log`      | String | 错误日志内容。                                                                      |
<!-- markdownlint-enable -->

> [!NOTE]
>
> `show_device_info` 仅在根因节点为非 Unknown Device 或存在首错节点时输出，且仅在首个根因节点或首错节点存在 error 日志时包含具体字段。

- Knowledge_Graph 字段

<!-- markdownlint-disable -->
| 字段              | 类型         | 说明                                          |
|-------------------|--------------|-----------------------------------------------|
| `analyze_success` | Boolean      | 故障事件分析是否成功，true 成功，false 失败。 |
| `version_info`    | Object       | 版本信息。                                    |
| `note`            | String       | 备注。                                        |
| `fault`           | List[Object] | 故障事件列表，字段见下表。                    |
<!-- markdownlint-enable -->

- fault 字段

<!-- markdownlint-disable -->
| 字段             | 类型         | 说明                                                         |
|------------------|--------------|--------------------------------------------------------------|
| `code`           | String       | 故障码。                                                     |
| `component`      | String       | 故障组件。                                                   |
| `module`         | String       | 故障模块。                                                   |
| `cause_zh`       | String       | 故障原因（中文）。                                           |
| `description_zh` | String       | 故障描述（中文）。                                           |
| `suggestion_zh`  | List[String] | 故障建议（中文）。                                           |
| `class`          | String       | 故障类别。                                                   |
| `event_attr`     | Object       | 故障事件属性，key 为设备名称，value 为事件属性列表，见下表。 |
| `fault_source`   | List[String] | 故障来源设备列表。                                           |
| `fault_chains`   | List         | 故障传播链。                                                 |
<!-- markdownlint-enable -->

- event_attr 事件属性字段

`event_attr` 的 value 为事件属性对象列表，每个对象包含以下字段：

<!-- markdownlint-disable -->
| 字段              | 类型               | 说明                                               |
|-------------------|--------------------|----------------------------------------------------|
| `event_code`      | String             | 事件故障码。                                       |
| `key_info`        | String             | 关键日志信息。                                     |
| `is_custom_event` | Boolean            | 是否为自定义事件。                                 |
| `source_device`   | String             | 来源设备。                                         |
| `source_file`     | String             | 来源日志文件名。                                   |
| `occur_time`      | String             | 故障发生时间。                                     |
| `type`            | String             | 日志类型。                                         |
| `occurrence`      | List[List[String]] | 故障发生记录列表，每条记录为 [发生时间, 关键日志]。 |
| `event_id`        | String             | 事件 ID。                                          |
<!-- markdownlint-enable -->

- ascend-fd 运行错误码请查阅[组件错误码](../07_references/04_appendix.md#组件错误码)。
