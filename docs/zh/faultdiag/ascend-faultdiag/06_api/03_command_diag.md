# diag命令（故障诊断）

## 功能说明

用户分析节点故障根因，如需分析多节点的故障根因，诊断前需要将各节点的清洗结果汇总到同一目录下，目录格式请参考[参数说明](#参数说明)。

## 命令格式

```shell
ascend-fd diag [-h] -i INPUT_PATH -o OUTPUT_PATH [-p] [-s {host,super_pod}]
```

## 参数说明

| 参数              | 类型   | 必选 | 说明                                                                   |
|-------------------|--------|------|------------------------------------------------------------------------|
| -h, --help        | -      | 否   | 显示帮助信息                                                           |
| -p, --performance | -      | 否   | 执行所有诊断模块（含设备资源分析和网络拥塞分析）                       |
| -i, --input_path  | string | 是   | 清洗完毕数据输入路径                                                   |
| -o, --output_path | string | 是   | 诊断结果输出路径                                                       |
| -s, --scene       | string | 否   | 诊断场景，可选值：host（主机场景）、super_pod（超节点场景），默认 host |

清洗完毕数据输入路径目录结构

```text
诊断输入目录
├── 节点1清洗结果/
│   ├── ascend-rc-parser.json
│   ├── ascend-kg-analyzer.json
│   ├── ascend-kg-parser.json
│   └── ...
├── 节点2清洗结果/
│   └── ...
└── 节点N清洗结果/
    └── ...
```

> [!NOTE]
>
> - 多机清洗数据，需手动将每个节点的清洗结果合并到一个目录下。
> - 单机清洗数据结构，请参考节点1清洗结果，字段意义请参考[清洗结果说明](02_command_parse.md#清洗结果说明)。

├── fault_diag_result

> - 回显未诊断出根因节点且故障事件分析无结果，才会执行设备资源分析和网络拥塞分析，表示当前故障属于性能劣化问题，不会导致训练任务异常退出。
> - 部分关键参数仅在特定场景中存在。

## 注意事项

- 诊断前需要确保各节点的系统时间已同步
- 集群规模超过 128 台服务器（1024 卡）时，需使用 `ulimit -n` 调整文件描述符上限
- ascend-fd 运行错误码请查阅[参考 -> 常用操作 -> 组件错误码](../07_references/04_appendix.md#组件错误码)
