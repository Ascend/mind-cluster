# 超节点故障诊断

超节点故障诊断是专门针对 Atlas A3 超节点（SuperPoD）场景的诊断功能，需要同时考虑 Host（服务器）、BMC（管理控制器）、LCNE（灵衢交换机）这三类日志。

> Host 日志即为[采集日志归档目录结构](02_log_collection.md#采集日志归档目录结构)中除 BMC 和 LCNE 以外的所有日志。

## 使用场景

超节点故障诊断支持以下两种场景：

| 场景             | 适用条件                      | 说明                       |
|------------------|-------------------------------|----------------------------|
| 拓扑信息自动关联 | Host、BMC、LCNE 三类日志齐全  | 最简单的方式，自动关联拓扑 |
| 拓扑信息手动关联 | 缺少 Host、BMC、LCNE 任一日志 | 需要手动关联清洗结果       |

> 注意：日志级别配置较低时，可能存在日志刷屏冲掉关键日志导致无法诊断的情况，请合理配置日志级别。

## 场景一：拓扑信息自动关联（推荐）

**前提：** Host、BMC、LCNE 三类日志齐全，该场景下诊断命令需要添加 `-s super_pod` 参数。

1. 执行清洗操作

    示例：

    ```shell
    ascend-fd parse -i <设备0上Host日志目录> -o <超节点清洗结果输出目录/host/worker-0>
    ascend-fd parse --lcne_log <设备0上LCNE日志目录> -o <超节点清洗结果输出目录/lcne/worker-0>
    ascend-fd parse --bmc_log <设备0上BMC日志目录> -o <超节点清洗结果输出目录/bmc/worker-0>
    ```

    > [!NOTE]
    >
    > - Host 侧日志必须包含 dmidecode.txt 信息。
    > - 如有多个设备，相应清洗到 worker-1、worker-2 等目录，也可用设备 ip 命名。

2. 清洗后的目录结构

    ```text
    超节点清洗结果输出目录/
        ├── bmc
        |   └── worker-0
        |       ├── ascend-kg-analyzer.json
        |       ├── ascend-kg-parser.json
        |       └── server-info.json
        ├── host
        |   └── worker-0
        |       ├── ascend-kg-analyzer.json
        |       ├── ascend-kg-parser.json
        |       ├── ascend-rc-parser.json
        |       └── server-info.json
        └── lcne
            └── worker-0
                ├── ascend-kg-analyzer.json
                ├── ascend-kg-parser.json
                └── server-info.json
    ```

3. 执行诊断（需添加 `-s super_pod` 参数）

    ```shell
    ascend-fd diag -i <诊断输入目录> -o <诊断结果输出目录> -s super_pod
    ```

## 场景二：拓扑信息手动关联

当缺失 Host、BMC、LCNE 任一日志时，需要手动关联将同一个节点的数据清洗到同一个目录。

1. 执行清洗操作

    ```shell
    # 清洗时同时指定 Host、BMC 日志
    ascend-fd parse --process_log <设备0上plog日志目录> --bmc_log <设备0上BMC日志目录> -o <超节点清洗结果输出目录/worker-0>
    ascend-fd parse --process_log <设备1上plog日志目录> --bmc_log <设备1上BMC日志目录> -o <超节点清洗结果输出目录/worker-1>
    ascend-fd parse --process_log <设备2上plog日志目录> --bmc_log <设备2上BMC日志目录> -o <超节点清洗结果输出目录/worker-2>
    ```

    > [!NOTE]
    >
    > - 示例为假设缺少 LCNE 日志。
    > - Host 侧日志以 --process_log 为例，可以直接使用 -i 把其他收集的日志包括进来（如 device_log、dl_log 等）。
    > - 不同设备的清洗结果，应归属于不同的 worker 下。
    > - 不同的设备，应该对应不同的 worker 目录（如五台设备，分别对应 worker-0 至 worker-4）。
    > - 示例仅展示三台设备的清洗结果，实际场景中，需要根据设备数量，对应不同的 worker 目录。

2. 清洗完成后的目录结构

    ```text
        超节点清洗结果输出目录/
        ├── worker-0
        │   ├── ascend-kg-analyzer.json
        │   ├── ascend-kg-parser.json
        │   ├── ascend-rc-parser.json
        │   ├── device_ip_info.json
        │   ├── plog-parser-14121-1.log
        │   └── server-info.json
        ├── worker-1
        │   ├── ascend-kg-analyzer.json
        │   ├── ascend-kg-parser.json
        │   ├── ascend-rc-parser.json
        │   ├── device_ip_info.json
        │   ├── plog-parser-14139-1.log
        │   └── server-info.json
        └── worker-2
            ├── ascend-kg-analyzer.json
            ├── ascend-kg-parser.json
            ├── ascend-rc-parser.json
            ├── device_ip_info.json
            ├── mindie-cluster-info.json
            ├── plog-parser-14160-1.log
            └── server-info.json
    ```

3. 执行诊断（不需要`-s super_pod` 参数）

    ```shell
    ascend-fd diag -i <诊断输入目录> -o <诊断结果输出目录>
    ```
