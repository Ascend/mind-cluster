# 简介

## 概述

MindCluster Ascend FaultDiag Toolkit（简称 ascend-fd-tk）是面向昇腾 AI 集群的链路故障诊断工具。工具提供交互式与非交互式两种操作模式，具备在线数据采集与离线日志分析两种数据处理能力，可完成集群设备信息采集与自动化巡检诊断，精准定位服务器、L1/L2 灵衢交换机、RoCE 交换机及 BMC 链路故障。

## 适用场景

ascend-fd-tk 适用于以下场景：

- **业务故障定位**：训练 / 推理任务异常时，快速采集日志并定位链路故障点。
- **多网络平面统一诊断**：支持多网络平面场景下的分批采集与汇总统一诊断。
- **客户定制化巡检**：按需自定义巡检维度，满足客户特定场景的链路巡检需求。
- **集群周期巡检**：支撑集群日常巡检，提前排查链路潜在异常。

## 核心能力

| 能力 | 说明                                                                 |
|------|--------------------------------------------------------------------|
| 在线采集与清洗 | 根据配置的集群设备（服务器、BMC、交换机）连接信息（账号、密码 / 密钥 / 免密），自动通过 SSH 采集日志并完成日志信息清洗 |
| 离线日志清洗 | 导入已采集的服务器带内日志、BMC dump 日志和交换设备 diagnostic information 日志，进行信息清洗    |
| 一键诊断 | 自动完成信息采集与故障诊断，生成诊断报告                                               |
| 分批采集与统一诊断 | 支持多网络平面场景下分批采集设备信息，汇总后统一诊断                                         |
| 客户定制化巡检 | 支持按客户类型定制诊断规则进行巡检                                                  |

## 操作系统

ascend-fd-tk 支持以下操作系统：

| 操作系统 | 版本要求 | 架构 |
|----------|----------|------|
| Linux | CentOS 7.6+ / Ubuntu 18.04+ / OpenEuler 20.03+ | x86_64 / aarch64 |
| Windows | Windows 10 / Windows 11 | x86_64 |

## 使用流程

用户可参照如下链路故障诊断流程完成故障诊断操作：

1. **安装工具**：通过 WHL 包完成工具安装。
2. **配置文件设置**（可选）：提供机房位置等附加配置信息。
3. **配置数据源 / 设置日志路径**：在线模式配置连接文件；离线模式设置日志目录。
4. **日志清洗**：在线模式自动收集设备日志并清洗；离线模式直接清洗日志信息。
5. **诊断 / 巡检**：对清洗后的数据进行故障诊断或巡检。
6. **查看报告**：诊断 / 巡检完成后，报告自动生成至家目录下。

> 每次使用工具前，建议先执行 `clear_cache` 清理缓存，以避免上一次诊断任务的残留数据影响本次诊断结果。
