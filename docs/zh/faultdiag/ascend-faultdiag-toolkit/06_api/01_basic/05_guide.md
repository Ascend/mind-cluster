# guide

## 命令功能

获取诊断工具的使用向导信息。首次使用建议先执行该命令了解整体流程。

## 命令格式

| 命令格式 | 描述 |
|---------|------|
| `guide` | 获取向导信息 |
| `guide ?` | 查看详情 |

## 参数说明

无业务参数，`?` 为内置帮助标识，用于查看命令用法。

## 输出说明

控制台输出三段式向导：① 采集内容准备（在线 / 离线 / 默认路径）→ ② 启动采集 / 分析与诊断 → ③ 清理缓存。

## 示例

非交互式方式（展示命令与回显）：

```bash
ascend-fd-tk  guide
        一、采集内容准备
        请根据故障设备自行按需选择要采集的设备信息或需要导入的日志，可以不导入全量设备信息或日志。按需设置以下在线或离线采集分析的任意地址。

        1. 在线采集准备
        若需要在线采集设备信息，请使用 " set_conn_config " 命令设置设备信息，具体配置可使用" set_conn_config ? "查看详情

        2. 离线日志解析准备
        2.1 设置服务器日志目录地址
        请使用 " set_host_dump_log " 命令设置离线日志目录，具体配置可使用" set_host_dump_log ? "查看详情

        2.2 设置BMC日志目录地址
        请使用 " set_bmc_dump_log " 命令设置离线日志目录，具体配置可使用" set_bmc_dump_log ? "查看详情

        2.3 设置交换机回显文本目录地址
        请使用 " set_switch_dump_log " 命令设置离线日志目录，具体配置可使用"set_switch_dump_log ? "查看详情

        3. 默认读取路径
        当未手动设置以上文件或目录时，工具会自动读取执行路径下的以下默认文件或目录，相关文件或目录需用户提前手动创建：
        连接配置: conn.ini
        BMC日志目录: bmc_dump_log
        Host日志目录: host_dump_log
        交换机日志目录: switch_dump_log

        二、启动采集/分析 & 诊断
        执行 " auto_collect_diag " 启动在线采集/离线分析并诊断

        三、清理缓存
        本工具支持分批采集统一诊断，所以单次诊断完后会留有缓存，若已完成诊断任务，请使用 " clear_cache " 清理缓存(若无法有效清理，请使用管理员模式打开工具)，避免影响下次诊断结果

        总结:
        1. 先用 " set_conn_config " 设置要访问的设备ip配置文件或用 " set_bmc_dump_log "，" set_host_dump_log "，" set_switch_dump_log "设置离线日志目录，或直接将日志放到默认目录下
        2. 以上至少有一项设置存在即可使用 " auto_collect_diag " 采集/分析并诊断输出报告
```

交互式方式（展示命令与回显）：

```bash
ascend-fd-tk
>>> guide
# 回显结果同上
```
