# auto_collect

## 命令功能

启动自动信息采集，支持离线、在线采集，适用于不同网络平面分批收集。

| 特性 | 说明                                             |
|------|------------------------------------------------|
| 采集模式 | 支持离线采集和在线采集                                    |
| 数据源 | 根据已配置的数据源（在线连接配置或离线日志目录），自动完成日志采集和数据清洗         |
| 适用场景 | 适用于不同网络平面分批收集（可多次执行 `auto_collect`，采集不同批次设备信息） |
| 后续步骤 | 收集完成后需对清洗后的结构化数据进行诊断或者巡检              |

## 命令格式

| 命令格式 | 描述 |
|---------|------|
| `auto_collect` | 启动自动信息采集，支持离线、在线采集，适用于不同网络平面分批收集 |
| `auto_collect ?` | 查看详情 |

## 参数说明

无业务参数，`?` 为内置帮助标识，用于查看命令用法。

## 数据源要求

执行 `auto_collect` 前需确保至少满足以下条件之一：

- 已通过 `set_conn_config` 配置在线设备连接信息。
- 已通过 `set_host_dump_log` / `set_bmc_dump_log` / `set_switch_dump_log` 设置离线日志目录。

## 输出说明

命令执行成功时返回：`收集完成，若完成全部收集请进行诊断/巡检`。

## 示例

非交互式方式（展示命令与回显）：

```bash
ascend-fd-tk set_conn_config /home/user/conn.ini auto_collect
# 其他日志输出...
收集完成，若完成全部收集请进行诊断/巡检
```

交互式方式（展示命令与回显）：

```bash
ascend-fd-tk
>>> auto_collect
收集完成，若完成全部收集请进行诊断/巡检
```
