# auto_collect_diag

## 命令功能

启动一键式自动收集诊断。自动执行收集（在线设备采集或离线日志收集）和诊断流程，等价于依次执行 `auto_collect` + `auto_diag`。

**适用场景**：设备可直接访问或日志已就绪，希望一步完成端到端诊断的场景。适用于单网络平面一次性诊断，如果需要对多网络平面分批采集，请使用 `auto_collect` + `auto_diag` 分步执行。

## 命令格式

| 命令格式 | 描述 |
|---------|------|
| `auto_collect_diag` | 启动一键式自动收集和诊断。 |
| `auto_collect_diag ?` | 查看详情。 |

## 参数说明

无参数。

## 执行流程

执行 `auto_collect_diag` 后，工具内部按以下步骤自动完成诊断：

1. **自动采集**：根据已配置的数据源（在线连接 / 离线日志目录），采集设备日志和状态信息。
2. **数据清洗**：解析原始日志，提取结构化数据并存入缓存。
3. **故障诊断**：对清洗后的数据运行全部诊断项，定位故障。
4. **生成报告**：输出 Excel 格式诊断报告至家目录下的 `report/` 子目录。

## 输出说明

- 成功：`诊断完成`
- 报告生成失败：`生成报告失败，解除占用后，可使用 'auto_diag' 重新生成报告。`

## 示例

非交互式方式：

```bash
ascend-fd-tk set_conn_config /home/user/conn.ini auto_collect_diag
# 其他日志输出...
诊断完成
```

交互式方式：

```bash
ascend-fd-tk
>>> auto_collect_diag
诊断完成
```
