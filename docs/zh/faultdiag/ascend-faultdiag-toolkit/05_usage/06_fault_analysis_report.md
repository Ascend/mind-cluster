# 诊断 / 巡检报告说明

本文档介绍 ascend-fd-tk 诊断报告和巡检报告的输出位置、文件格式与字段含义，用于解读诊断 / 巡检结果。

## 报告说明

| 报告项   | 说明                                                                                                                 |
|-------|--------------------------------------------------------------------------------------------------------------------|
| 报告目录  | Linux: `~/.ascend-faultdiag-toolkit/report/`<br>Windows: `{当前工作目录}/.ascend-faultdiag-toolkit/report/`              |
| 报告文件名 | 诊断报告：`diag_report_{YYYYMMDD_HHMMSS}.xlsx`（如 `diag_report_20260629_143022.xlsx`）  <br/>巡检报告：`inspection_errors.csv` |
| 报告格式  | 诊断报告：Microsoft Excel（`.xlsx`），包含多个 Sheet <br/> 巡检报告：CSV 格式                                                         |
| 报告字符集 | UTF-8（中文描述正常显示）                                                                                                    |

> **快速导航**：[诊断报告](#诊断报告) | [巡检报告](#巡检报告)

### 故障诊断 Sheet 字段

#### 带内故障分析(Host)  Sheet

| 字段 | 含义 |
|------|------|
| 主机ID | 主机在工具内部的唯一标识 |
| 主机名 | 主机 `hostname` 命令的输出 |
| SN | 主机序列号 |
| 机房名称 | 来自 LLD.xlsx 配置文件 |
| 机柜编号 | 来自 LLD.xlsx 配置文件 |
| NPU ID | NPU 编号 |
| 物理芯片ID | 物理芯片 ID（chip_phy_id）|
| 故障码 | 工具内部定义的故障代码（部分诊断项为空）|
| 故障信息 | 故障的详细描述（含数据值与阈值）|
| 处理建议 | 推荐的处置方式 |
| 是否链路故障根因 | 是否为本次诊断的根因（是/否/unknown）|

#### 带外故障分析(BMC)  Sheet

| 字段 | 含义 |

| NPU ID | NPU 编号 |
| 物理芯片ID | 物理芯片 ID |

| 字段 | 含义 | 示例 |

### 光模块信息 Sheet 字段

#### 主机NPU\<-\>RoCE交换机端口光模块信息 Sheet

该 Sheet 采用「主机侧」+「交换机侧」两段式布局，每侧均包含 4 个 Lane（Lane 0 ~ Lane 3）的详细光模块指标。

- **主机侧字段**

| 字段类别 | 字段 |
|---------|------|
| 设备信息 | 主机ID、主机名、主机SN、机房名称、机柜编号 |
| 芯片信息 | NPU ID、芯片ID、物理芯片ID、NPU类型、主机侧端口 |
| 光模块基本信息 | 主机侧光模块状态、光模块厂商、光模块型号、光模块SN、光模块温度、光模块供电电压 |
| 链路状态 | 主机侧链路速度、双工模式、网络健康、链路状态、链路UP次数、链路DOWN次数 |
| Lane 指标（Lane 0~3） | TX Power、RX Power、TX Bias、Host SNR、Media SNR |

- **对端交换机侧字段**

| 字段类别 | 字段 |
|---------|------|
| 设备信息 | 对端交换机名称、对端交换机ID、对端交换机SN、机房名称、机柜编号 |
| 端口信息 | 对端交换机端口、端口速度、端口双工模式、端口状态 |
| 光模块基本信息 | 对端交换机光模块厂商、光模块型号、光模块SN、光模块温度 |
| Lane 指标（Lane 0~3） | TX Power、RX Power、SNR |

#### 交换机间端口连接光模块信息 Sheet

| 设备信息 | 对端交换机名称、交换机ID、交换机SN、机房名称、机柜编号 |

### 报告解读流程

#### 步骤 1：定位关键 Sheet

按故障类型聚焦：

| 故障现象 | 优先查看 |
|----------|----------|
| 主机 NPU 训练/推理异常、丢包、延迟高 | **带内故障分析(Host)** |
| BMC 告警灯亮、无法登录 BMC | **带外故障分析(BMC)** |
| 节点间网络不通、丢包严重、闪断 | **交换机故障分析(L1&L2&RoCE)** |
| 机房级别故障（多机柜多节点）| 优先看交换机 Sheet，再看 Host/BMC |
| 链路映射关系核对 | **NPU&CPU到L1链路分析** / **L1到L2链路分析** |
| 光模块状态核对 | **主机NPU\<-\>RoCE交换机端口光模块信息** / **交换机间端口连接光模块信息** |

#### 步骤 2：筛选根因

在故障诊断 Sheet 的「是否链路故障根因」列筛选值为「是」的记录，这些是工具识别的本次诊断的根因故障。

> - 标记为「否」的记录表示该故障可能是其他故障的衍生（如「端口 down」是因为「对端交换机电源故障」），可作为参考但非首要处理项。
> - 标记为「unknown」的记录表示工具无法判定是否为根因，需要人工结合业务场景判断。

#### 步骤 3：按处置建议执行

在「处理建议」列查看工具推荐的处置方式。

#### 步骤 4：核对链路与光模块信息

在链路分析 Sheet 与光模块信息 Sheet 中查看详细的链路映射关系与光模块指标，对比异常端口与本端/对端数据。

## 巡检报告

执行 `auto_inspection` 命令后，工具会生成巡检报告 `inspection_errors.csv`，记录本端设备与对端设备之间端口连接的光模块异常信息。与诊断报告的 Excel 格式不同，巡检报告为 **CSV 格式**，可使用文本编辑器或电子表格软件（如 Microsoft Excel、WPS 表格）打开查看。

### 巡检报告字段

| 字段 | 含义 |
|------|------|
| A端设备名称 | 本端设备名称 |
| A端IP | 本端设备 IP 地址 |
| A端接口 | 本端设备物理端口标识 |
| A端SN | 本端设备序列号 |
| A端光模块SN | 本端端口光模块序列号 |
| B端设备名称 | 对端设备名称 |
| B端IP | 对端设备 IP 地址 |
| B端接口 | 对端设备物理端口标识 |
| B端SN | 对端设备序列号 |
| B端光模块SN | 对端端口光模块序列号 |
| 问题现象 | 巡检发现的异常描述（如光模块功率异常、SNR 过低、链路状态异常等） |

> 巡检报告中的每一行代表一对存在异常的端口连接，A 端为本端、B 端为对端，便于快速定位两端设备对应的光模块问题。

### 巡检报告解读

巡检报告主要用于快速发现集群中光模块层面的物理连接异常。建议按以下步骤解读：

1. **筛选高频问题**：按「问题现象」列排序或筛选，快速定位集中出现的故障类型。
2. **定位具体设备**：根据「A端设备名称」「A端IP」「A端接口」定位本端异常端口，再结合「B端设备名称」「B端IP」「B端接口」确认对端信息。
3. **核对光模块信息**：使用「A端光模块SN」「B端光模块SN」查询到对应光模块，必要时更换光模块。
