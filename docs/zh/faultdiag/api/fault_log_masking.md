# 屏蔽故障日志接口<a name="ZH-CN_TOPIC_0000001799772104"></a>

**接口原型<a name="zh-cn_topic_0000001511538701_section124882040143613"></a>**

```
ascend-fd blacklist 子命令
```

**功能说明<a name="zh-cn_topic_0000001511538701_section12230185113815"></a>**

新增含有故障关键词的屏蔽规则，实现在日志清洗时不将含有故障关键词的信息记录到日志清洗后的文件中。

>[!NOTE] 说明 
>-   当前仅支持对CANN应用类日志的ERROR日志进行屏蔽操作。
>-   若用户需要自定义屏蔽故障关键词文件的保存路径，可以参考[自定义MindCluster Ascend FaultDiag家目录](../common_operations.md#自定义mindcluster-ascend-faultdiag家目录)章节进行操作。

**参数说明<a name="zh-cn_topic_0000001511538701_section122149111390"></a>**

**表 1**  子命令参数说明

|**参数**|**缩写**|是否必选|**值类型**|**说明**|
|--|--|--|--|--|
|--add|-a|必选，且参数间互斥，即只能指定一个且必须指定一个参数。|String|新增包含关键词的屏蔽规则。<ul><li>关键词支持的最大取值长度为200个字符，支持大小写字母、数字和其他字符（如-./等）。</li><li>一条屏蔽规则最多支持10个关键词，支持通过空格分隔多个关键词，如ERROR FUSION、ERROR "FUSION"或"ERROR" "FUSION"。</li><li>如果关键词包含“\”，为避免“\”被忽略，整个关键词须打引号，如“ERR\OR”。</li><li>当前最多保存50条屏蔽规则。若超出50条，系统将丢弃最前面的屏蔽规则，保留新增的屏蔽规则。</li></ul>|
|--file|-f|必选，且参数间互斥，即只能指定一个且必须指定一个参数。|String|导入屏蔽规则JSON文件。使用JSON文件的方式新增屏蔽规则，导入的JSON文件会覆盖系统原有的JSON文件中的内容。|
|--delete|-d|必选，且参数间互斥，即只能指定一个且必须指定一个参数。|Int|删除屏蔽规则。支持使用空格分隔，同时删除多条规则。|
|--show|-s|必选，且参数间互斥，即只能指定一个且必须指定一个参数。|String|查看当前已有的屏蔽规则。|
|--force|无|可选|Bool|指定该参数删除屏蔽规则或替换文件时，界面将不会出现确认提示信息。<p>必须与--delete或--file一起使用。</p>|
|--help|-h|可选|-|查询使用说明。|


**返回说明<a name="zh-cn_topic_0000001511538701_section1714345618323"></a>**

例如返回当前已有的屏蔽规则。

```
[BLACKLIST]
0. ERROR2, ERROR3, ERROR3
1. ERR1, ERR2, ERR3, ERR4
```
